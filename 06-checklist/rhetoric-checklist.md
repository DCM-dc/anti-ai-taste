# 06-checklist / Rhetoric

> Quick-reference leaf-node checklist. Run after producing argumentative
> or long-form output. This catches defaults; the principle layer
> (`04-rhetoric/`) prevents them.

---

## Quotation

- [ ] Count of quotes per 1000 words: ≤ 2.
- [ ] No Einstein / Jobs / Gandhi / Mandela / Oprah / Churchill quotes unless explicitly justified.
- [ ] No Chinese template quotes ("古有 X，今有 Y" / "忆往昔…看今朝…思未来…").
- [ ] No three-quote parallel ("X 说过…Y 也指出…Z 进一步强调…").
- [ ] Every quote is load-bearing (removing it would weaken the argument).
- [ ] After every quote, the writer analyzes it (not just "moves to next paragraph").
- [ ] Every statistic names the source (study name, year, sample size).
- [ ] No "studies show" / "research indicates" without naming the source.
- [ ] No hallucinated quotes, studies, or citations. If unsure, mark [source needed] or remove.
- [ ] For each quote: could you paraphrase it without losing meaning? If yes, paraphrase.

## Summarization

- [ ] Paragraph-level summaries: < 30% of paragraphs end with "in short," "therefore," "thus," "as a result."
- [ ] No "In conclusion" / "综上所述" / "To sum up" paragraph unless explicitly justified.
- [ ] No "Key Takeaways" bullet list unless user explicitly asked for TL;DR.
- [ ] Last sentence of the piece: could it end any article on any topic? If yes, replace with something specific.
- [ ] Forced-elevation phrases scanned for and removed:
      - "The future belongs to..."
      - "In this era of..."
      - "This reminds us that..."
      - "At the end of the day..."
      - "在这个……的时代"
      - "具有重要意义"
      - "Ready to transform your X? Try Y today."
- [ ] Delete-the-last-sentence test run on each paragraph. Decorative summaries deleted.
- [ ] Sandwich structure (intro / body / conclusion of equal weight) justified, or reshaped to: inverted pyramid / funnel / reverse funnel / thread / list.
- [ ] Ending leaves at least one door open (not all doors closed).

## Concept-coining

- [ ] Count of coined terms per 1000 words: ≤ 2.
- [ ] For each coined term: can you describe the concept in one plain sentence? If yes, delete the coinage.
- [ ] No "X's Law" without stating the equation or falsifiable prediction.
- [ ] No "The X Model" without defining inputs, outputs, and relationships.
- [ ] No "The X Framework" without testable predictions.
- [ ] No "The X Index" without formula and data source.
- [ ] No acronyms coined unless the concept appears so often that the full name is unwieldy.
- [ ] No redefining existing words as proprietary terms ("Skill" = Prompt+Tool bundle, etc.).
- [ ] No symmetric naming chosen for aesthetics (Five Intelligences, 5E Cascade, Seven Pillars).
- [ ] No letter-pattern naming (5E: Evolution, Effectiveness, Excellence, Ecosystems, Empowerment).
- [ ] Would this term appear in 1000 other AI "thought leadership" posts? If yes, don't coin it.

## Authority without substance

- [ ] No "augmented," "next-generation," "transformative," "paradigm-shifting" without specific mechanism.
- [ ] No "harness the power of," "unlock the potential of," "leverage the power of" without specific action.
- [ ] No "in today's fast-paced world," "in the modern era," "in the digital age" — these are pure decoration.
- [ ] No "comprehensive solution," "end-to-end platform," "enterprise-grade" without specific feature list.

## Self-awareness

- [ ] The text admits at least one limitation, weakness, or open question.
- [ ] The text does not claim to "solve" anything it can't demonstrate solving.
- [ ] Generalizations are scoped ("for our 12-person team," "in this specific case"), not universal.
- [ ] Where the writer doesn't know, they say so ("I don't know yet. Ask me in six months.").

## Final

- [ ] Read the piece as a skeptical reader. Where does it sound like "deep nothing"? Fix or remove.
- [ ] Could this entire piece appear in 1000 other AI "thought leadership" posts? If yes, rewrite with specifics.
- [ ] Does the piece have a fingerprint — something that could only have been written by this specific
      author about this specific topic? If no, add specificity until it does.
